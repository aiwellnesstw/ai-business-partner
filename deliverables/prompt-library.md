# AI 業務夥伴 — Prompt 庫（50 組）

> 使用方式：複製 Prompt，填入 `【】` 括號內的資訊，貼進 Claude 或 ChatGPT 即可。

---

## 一、陌生開發（10 組）

**P-01 開發前背景研究**
```
我要拜訪【公司名稱】，他們是做【行業】的，請幫我整理：
1. 這個行業目前面臨的 3 個主要痛點
2. 我的產品【產品/服務名稱】可以解決哪個痛點
3. 開場白建議（30 秒內說完）
```

**P-02 電話開場白生成**
```
幫我寫一段 B2B 電話開場白，情境如下：
- 我的身份：【你的姓名/公司】
- 對方：【職稱，例如：採購主管/老闆】
- 我的產品：【產品/服務】
- 我能幫他解決的問題：【痛點】
要求：30 秒內說完，不要太像推銷，要像在幫他解決問題。
```

**P-03 陌生 LINE 訊息（第一次聯絡）**
```
幫我寫一段陌生開發 LINE 訊息：
- 對象：【職稱/行業】
- 我的產品：【產品/服務】
- 切入點（為什麼要聯絡他）：【原因，例如：看到你們在招業務/參加了某展覽】
要求：3 句話以內，有吸引力，結尾有明確的行動呼籲。
```

**P-04 Email 開發信**
```
幫我寫一封 B2B 開發 Email：
- 收件人：【公司名稱】【職稱】
- 主旨：【你來想，要讓人想點開】
- 我的產品：【產品/服務】
- 我能帶來的具體價值：【例如：幫他省 30% 採購成本】
- 期望的下一步：【例如：安排 20 分鐘線上說明】
格式：主旨 + 正文，正文 150 字以內，語氣專業但不疏遠。
```

**P-05 社群媒體 (LinkedIn/Facebook) 開發訊息**
```
幫我寫一段 Facebook/LinkedIn 私訊，陌生開發用：
- 對方背景：【職稱/公司/行業】
- 我想聊的主題：【產品/服務/合作機會】
- 我看到他的什麼讓我想聯絡：【例如：看到你發的貼文/你們剛得獎】
要求：輕鬆自然，不要一上來就推銷，要先建立連結。
```

**P-06 展會後跟進訊息**
```
我在【展會名稱】認識了【對方姓名/職稱】，幫我寫一封展後跟進訊息：
- 我們聊過的話題：【話題】
- 我能提供的後續價值：【例如：寄報告/安排 demo】
語氣：像朋友，記得我們聊過的事，不要太正式。
```

**P-07 轉介紹首次聯絡**
```
【介紹人名字】介紹我去聯絡【對方姓名/公司】，幫我寫第一次聯絡的訊息：
- 介紹人與對方的關係：【關係】
- 我的產品/服務：【產品/服務】
- 期望下一步：【例如：約個線上說明】
要帶出介紹人的名字，讓對方有安全感。
```

**P-08 競爭對手客戶開發**
```
我知道【公司名稱】目前在用【競爭對手名稱】的產品，幫我寫一段開發訊息：
- 我的產品相比競爭對手的優勢：【優勢 1、2、3】
- 不要批評競爭對手，要強調我能多給什麼
語氣：自信但不攻擊，要讓對方覺得值得了解一下。
```

**P-09 老闆/決策者開發（繞過窗口）**
```
我一直被推給【窗口職稱】，沒辦法見到真正決策者，幫我寫一封直接給老闆/主管的信：
- 公司名稱：【公司】
- 老闆/主管職稱：【職稱】
- 我的產品能幫公司解決的問題：【問題】
要讓他覺得這件事值得他親自了解，而不是交給下面的人處理。
```

**P-10 冷名單重新激活**
```
這個客戶【公司名稱】上次接觸是【幾個月前】，當時沒有成交，原因是【當時的原因】。
現在幫我寫一段重新聯絡的訊息，切入點是【現在的新切入點，例如：有新方案/市場有變化】。
不要提上次失敗的事，要讓他覺得這次有新的理由值得再聊。
```

## 二、提案與報價（15 組）

**P-11 快速提案架構生成**
```
幫我生成一份提案架構，資訊如下：
- 客戶公司：【公司名稱】，行業：【行業】
- 客戶痛點：【痛點 1】【痛點 2】
- 我的產品/服務：【產品/服務名稱】
- 預算範圍（如果知道）：【金額】
請給我：標題、客戶現況分析、我的解決方案、預期效益、下一步行動，每段 3-5 句話。
```

**P-12 報價單說明文字**
```
我要寄報價單給【客戶公司】，幫我寫報價單前面的說明段落：
- 產品/服務項目：【項目】
- 報價金額：【金額】
- 特別說明（折扣/有效期限/付款條件）：【說明】
語氣要讓客戶覺得這個價格值得，不要只是列數字。
```

**P-13 競標提案重點摘要**
```
我要參加【公司/機關名稱】的採購評選，幫我寫一段提案重點摘要（口頭報告開場用）：
- 標案需求：【需求】
- 我的解決方案亮點：【亮點 1、2、3】
- 為什麼選我而不是競爭對手：【差異化】
時間：2 分鐘內說完，要讓評委留下印象。
```

**P-14 產品 Demo 前的引導說明**
```
Demo 前幫我寫一段引導說明，讓客戶知道等等要看什麼、為什麼這對他重要：
- 客戶痛點（他自己說過的）：【痛點】
- Demo 會展示的功能：【功能 1、2、3】
- 希望他看完有的感受：【例如：這個能省我很多時間】
要讓他帶著問題去看 Demo，而不是被動接受資訊。
```

**P-15 提案後摘要 Email**
```
今天跟【客戶公司】【職稱】開完提案會議，幫我寫一封會後摘要 Email：
- 我們達成的共識：【共識】
- 待確認的事項：【事項】
- 下一步和時間節點：【下一步】
語氣：專業有效率，讓對方覺得我做事靠譜。
```

**P-16 客製化方案說明信**
```
幫我寫一封說明客製化方案的 Email：
- 客戶需求（他說的）：【需求】
- 我為他做的客製調整：【調整 1、2、3】
- 這個調整帶來的好處：【好處】
強調「這是專門為你設計的」，不要讓他覺得是罐頭方案。
```

**P-17 多方案比較表說明**
```
我要給客戶看三個方案（基本/標準/進階），幫我寫每個方案的一句話說明，讓他快速抓到差異：
- 基本方案（【金額】）：功能是【功能】
- 標準方案（【金額】）：功能是【功能】
- 進階方案（【金額】）：功能是【功能】
要讓他自然往中間或進階方案走。
```

**P-18 ROI 計算說明**
```
幫我計算並說明投資報酬率給客戶看：
- 我的產品費用：【費用】
- 客戶目前的成本（人力/時間/損失）：【現況成本】
- 用了我的產品後能省下或賺到多少：【效益】
用數字說話，讓他覺得不買才是在虧錢。
```

**P-19 反對意見回應腳本（價格太貴）**
```
客戶說「價格太貴了」，幫我寫回應：
- 我的產品：【產品/服務】
- 報價：【金額】
- 能帶來的效益：【效益】
不要只是降價，要先讓他重新看到價值，再談彈性。
```

**P-20 反對意見回應腳本（要再比較看看）**
```
客戶說「我還要再比較看看其他家」，幫我寫回應：
- 我的產品優勢：【優勢】
- 競爭對手（如果知道）：【競爭對手】
要讓他說出他真正在意的點，而不是讓他拿著我的提案去問競爭對手。
```

**P-21 口頭報價後的確認訊息**
```
剛才電話口頭報了【金額】給【客戶姓名】，幫我寫一則 LINE 訊息做書面確認：
- 產品/服務：【項目】
- 報價：【金額】
- 有效期限：【日期】
- 下一步：【例如：明天寄正式報價單】
簡短，讓他有正式感但不要太正式。
```

**P-22 提案失敗後的感謝信**
```
這次提案沒有得標/成交，幫我寫一封給【客戶公司】的感謝信：
- 感謝他的時間和機會
- 不卑不亢，留下好印象
- 為下次合作留一扇門
不要哀求，要讓他記得我是個值得合作的人。
```

**P-23 追加銷售/升級方案建議**
```
現有客戶【公司名稱】目前用的是【基本方案】，幫我寫一封建議他升級的訊息：
- 他目前的使用情況（如果知道）：【情況】
- 升級後能多得到什麼：【新功能/效益】
- 升級費用：【費用差異】
要讓他覺得是我在幫他，不是在推銷。
```

**P-24 聯合提案/合作邀請**
```
我想邀請【公司名稱】一起聯合提案給【目標客戶】，幫我寫一封合作邀請信：
- 我能貢獻的：【我的部分】
- 對方能貢獻的：【對方的部分】
- 合作對雙方的好處：【好處】
語氣：平等合作，強調互補。
```

**P-25 政府採購規格建議書**
```
我要向【機關名稱】建議修改採購規格，讓我的產品更符合需求，幫我草擬建議書：
- 目前規格的問題/不足之處：【問題】
- 我建議的修改：【建議】
- 修改後對機關的好處：【好處】
語氣：專業、站在機關利益的角度，不要讓他覺得我只是想得標。
```

## 三、客戶跟進（15 組）

**P-26 報價後 3 天沒消息**
```
3 天前寄了報價給【客戶姓名/公司】，還沒有回應，幫我寫一則跟進訊息：
- 報價的產品：【產品】
- 報價金額：【金額】
不要催，要給他一個回覆的理由（例如提供新資訊或問他有沒有問題）。
```

**P-27 提案後 1 週沒動靜**
```
上週做完提案給【客戶公司】，一週了還沒消息，幫我寫跟進訊息：
要自然地問他目前評估的進度，順便提供一個有用的資訊讓他有回覆的動力。
```

**P-28 節日/時事切入暖身**
```
我想趁【節日/時事，例如：端午節/新年/某個產業大事件】重新聯絡【客戶姓名】，幫我寫一則暖身訊息：
- 我和他的關係（接觸過幾次，聊過什麼）：【關係】
- 想順帶提的產品/服務：【產品】（可以輕輕帶一下，不要硬推）
自然，不要讓他覺得只是藉口聯絡。
```

**P-29 客戶說「之後再說」的跟進**
```
上次客戶說「之後再聯絡」，過了【時間】，幫我寫重新聯絡的訊息：
- 上次談到哪裡：【進度】
- 現在切入的新理由：【例如：有新方案/有類似案例可以分享】
讓他覺得現在是好時機重新考慮。
```

**P-30 已讀不回（第一次）**
```
我傳了訊息給【客戶姓名】，他已讀但沒回，幫我寫一則輕鬆的跟進訊息：
- 之前傳的是：【訊息內容摘要】
不要讓他有壓力，給他一個簡單回覆的理由。
```

**P-31 已讀不回（第二次，換方向）**
```
【客戶姓名】已讀不回第二次了，幫我換個方向跟進：
- 之前一直在談的是：【產品/話題】
- 現在換成：先不提產品，只是保持聯繫或問他一個他可能有興趣的問題
讓他願意重新開口說話。
```

**P-32 直接拒絕後的回應**
```
客戶直接說「不需要」或「不感興趣」，幫我寫一則優雅的回應：
- 不要哀求，要接受他的拒絕
- 留下好印象，為未來鋪路
- 可以問他一個問題，了解他真正的顧慮（如果適合的話）
```

**P-33 對手報價比我低的應對**
```
客戶說競爭對手報價比我低【金額/百分比】，幫我寫回應：
- 我的產品優勢：【優勢】
- 我的服務 / 售後比競爭對手好在哪：【差異】
不要直接批評對手，要讓客戶自己算清楚「便宜的代價」。
```

**P-34 成交後的感謝訊息**
```
【客戶公司】剛確認成交/簽約，幫我寫一則感謝訊息：
- 成交的產品/服務：【產品】
- 下一步會做什麼：【例如：安排交機/開始導入】
讓他覺得選擇我是對的，期待接下來的合作。
```

**P-35 成交後轉介紹請求**
```
【客戶姓名】用了一段時間，感覺滿意，幫我寫一則請他幫我介紹的訊息：
- 他用的產品：【產品】
- 他給過的正面回饋（如果有）：【回饋】
要讓他覺得介紹朋友是在幫朋友，不是在幫我。
```

**P-36 客戶投訴/問題處理後的跟進**
```
【客戶公司】之前有個問題【問題描述】，我們已經處理好了，幫我寫跟進確認訊息：
- 確認問題已解決
- 重建信任
- 讓他知道以後有問題可以直接找我
語氣：負責任，讓他覺得我是個可靠的合作夥伴。
```

**P-37 長期沒聯絡的老客戶重新接觸**
```
【客戶公司】已經超過【時間】沒有往來，幫我寫一則重新接觸的訊息：
- 上次合作的內容：【合作內容】
- 現在重新聯絡的理由：【理由，例如：有新產品/看到他們最近的新聞】
不要讓他覺得你只有需要他時才聯絡。
```

**P-38 合約到期前的續約提醒**
```
【客戶公司】的合約【月份】到期，幫我寫續約提醒訊息：
- 目前合約內容：【內容】
- 續約方案（如有調整）：【調整】
- 如果提早續約的優惠：【優惠】
讓他覺得現在續是划算的選擇。
```

**P-39 年底結算感謝 + 明年預熱**
```
年底了，幫我寫一封給所有重要客戶的感謝信，順帶預熱明年：
- 今年合作的亮點：【亮點（可以填通用的）】
- 明年有什麼新方向/產品想跟他聊：【方向】
溫暖有誠意，不要像群發廣告。
```

**P-40 客戶回購/加購建議**
```
【客戶公司】已經買過【產品 A】，幫我建議他加購【產品 B】：
- 產品 A 和產品 B 的關聯：【關聯】
- 一起用的好處：【好處】
讓他覺得加購是自然的下一步，不是被推銷。
```

## 四、標案相關（5 組）

**P-41 標案資格確認函**
```
我要詢問【機關名稱】的【標案名稱】是否已開放廠商報名，幫我寫一封確認函：
- 我的公司資格：【資格簡述】
- 想確認的事項：【事項，例如：投標資格/截止日期/評分標準】
語氣：正式，展現專業。
```

**P-42 標案得標後感謝與啟動信**
```
我們得標了【機關名稱】的【標案名稱】，幫我寫一封給承辦人的啟動信：
- 合約金額：【金額】
- 下一步：【例如：確認開工會議時間/提交工作計畫】
正式但友善，讓他知道我們會把這個案子做好。
```

**P-43 未得標後的感謝與關係維護**
```
【機關名稱】的【標案名稱】這次沒得標，幫我寫感謝信給承辦人：
- 感謝評選機會
- 詢問落選原因（委婉地）
- 為下次機會鋪路
不能太明顯地問原因，要讓他願意分享。
```

**P-44 標案資料整理摘要**
```
幫我整理以下政府採購公告，提取重點讓我快速判斷要不要投標：
【貼上標案公告內容】

請整理：
1. 機關名稱與聯絡人
2. 標案金額與預算
3. 投標截止日期
4. 主要採購項目
5. 我們是否有資格 / 競爭力（根據【我的公司條件】判斷）
6. 建議：投 / 不投 / 需要更多資訊
```

**P-45 政府機關拜訪前的功課**
```
我下週要去拜訪【機關名稱】的【職稱/單位】，幫我做功課：
- 這個機關的主要業務：【機關業務】
- 我的產品如何切入他們的需求：【切入點】
- 他們近期可能有的採購需求（根據預算/業務推測）：【推測】
- 建議的話題和切入角度：【角度】
讓我去之前有充分準備，不要一問三不知。
```
